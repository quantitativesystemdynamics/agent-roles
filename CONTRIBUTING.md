# Contributing to Agent Roles

Thank you for your interest in improving this role library.

## How to Contribute

### Adding New Roles

1. Copy `role-template.md` to the appropriate domain directory
2. Fill in all required sections
3. Use kebab-case for filenames (e.g., `new-role-name.md`)
4. Include a clear **Purpose** statement in the first 50 lines

### Modifying Existing Roles

1. Ensure changes maintain backward compatibility
2. Update the purpose statement if the role's focus changes
3. Document significant changes in a changelog entry

### Domain Categories

If creating a new domain:
- `compliance/` — Regulatory compliance
- `data-privacy/` — Privacy engineering
- `executive/` — C-suite and strategy
- `finance/` — Finance and accounting
- `governance/` — Oversight and governance
- `it/` — IT operations
- `legal/` — Legal and contracts
- `operations/` — Operational support
- `people/` — HR and culture
- `reliability/` — SRE and reliability
- `security/` — Security engineering
- `trust-and-safety/` — Content moderation
- `weird-but-useful/` — Specialized cognitive roles

## Quality Standards

- Purpose statement: One sentence, max 200 characters
- Key responsibilities: 5-10 items
- Deliverables: Concrete, measurable outputs
- Cross-references: Use existing role names when applicable

## Questions?

Open an issue for discussion before submitting large changes.
